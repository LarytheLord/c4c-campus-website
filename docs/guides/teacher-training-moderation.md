## Module 1: Getting Started with Moderation

### What is Moderation?

**Moderation** means managing online discussions to keep them:
- On-topic and productive
- Respectful and safe
- High-quality and valuable
- Accessible to all students

**Your Role as Moderator:**
- Guide discussions toward learning goals
- Remove harmful or off-topic content
- Answer questions and clarify concepts
- Model respectful communication
- Create an inclusive learning environment

### Why Moderation Matters

**Good moderation:**
- Increases student engagement
- Reduces anxiety about asking questions
- Prevents harassment and bullying
- Maintains focus on learning
- Builds community
- Attracts quality discussions

**Without moderation:**
- Spam and ads clutter discussions
- Misinformation spreads
- Students feel unsafe
- Harassment goes unchecked
- Valuable answers get buried
- Trolls derail conversations

### Your Moderation Philosophy

**Before you start, consider:**

**What kind of community do I want?**
- Collaborative? Competitive?
- Formal? Casual?
- Large discussions? Focused threads?
- Student-led? Teacher-guided?

**My approach to student contributions:**
- Do I want to answer everything immediately?
- Should students answer each other first?
- How do I validate good student contributions?
- What's my response time expectation?

**How I'll handle violations:**
- What's my bar for deletion vs. editing?
- How will I respond to first-time violations?
- What's my escalation path for serious issues?
- How transparent will I be about moderation?

**Example Philosophy Statement:**
```
I will foster a collaborative learning community where:
- Students feel safe asking questions
- Peer learning is encouraged and celebrated
- Teachers provide guidance, not just answers
- Discussions stay focused on learning
- Violations are handled respectfully and fairly
```

## Module 3: Core Moderation Actions

### Action 1: Responding to Questions

**When a student asks a question:**

**Step 1: Read the question carefully**
- What are they really asking?
- What do they already know?
- What's the learning goal?

**Step 2: Decide your approach**
- Answer directly? Guide with questions?
- Let peers answer first?
- Pin this for future students?

**Step 3: Write your response**
- Be clear and concise
- Use examples
- Explain the "why"
- Show you care

**Your Response Template:**
```
Great question! [Acknowledgment]

Quick answer: [Direct response]

Here's the detail: [Explanation with reasoning]

Example: [Code example or scenario]

Common mistake: [What students often get wrong]

Try this and let me know!
```

**Example Response:**
```
Great question! Many students ask this.

To authenticate with the Google Sheets API in n8n:

1. Go to Google Cloud Console (https://console.cloud.google.com)
2. Create a new project
3. Enable Google Sheets API
4. Create OAuth 2.0 credentials
5. Download the JSON file
6. In n8n, add your credentials

More detail: [Video link]

Common mistake: Students forget step 3 (enable API)

Try it and let me know what error you see!
```

### Action 4: Deleting Content

**When to Delete:**
- Spam or advertisements
- Harassment or bullying
- Misinformation or scams
- Off-topic content
- Privacy violations
- Duplicates
- Student request

**Step 1: Click "Delete" button**
```
Find the post to delete
Click [Delete] button
```

**Step 2: Choose deletion type**
```
Soft delete: "[Removed by moderator]" shows (preserves context)
Hard delete: Completely gone (careful - no undo!)
```

**Step 3: Confirm deletion**
```
Click "Confirm Delete"
Post is removed
```

**After Deletion:**
- Consider messaging the student (if policy violation)
- Document the reason (for pattern tracking)
- Monitor for repeat violations

**Deletion Confirmation:**
```
[This discussion was removed by a moderator]

[Learn more about community guidelines]
```

**When to Use Each Type:**

**Soft Delete (Recommended):**
- Non-urgent violations
- Experimental posts
- Minor content issues
- Preserves thread context

**Hard Delete (Only When Necessary):**
- Spam/ads
- Hate speech
- Explicit content
- Privacy violations
- Illegal content

**Caution:** Hard delete also removes all child replies!

## Module 4: Common Scenarios

### Scenario A: Multiple Wrong Answers

**The Situation:**
```
Student A: "Use the HTTP Request node"
Student B: "Change your settings to [wrong thing]"
Student C: "That's not how it works"

[Confusion! Students need guidance]
```

**Your Action:**

**1. Don't delete (learning opportunity)**
```
Let the wrong answers stay - it's educational
```

**2. Reply to the original question**
```
"Great attempts, everyone! Let me clarify..."
```

**3. Explain why each approach has issues**
```
"@Student A: This node requires [specific setup]"
"@Student B: That won't work because [reason]"
```

**4. Provide the correct approach**
```
"The right way is:
1. [Step 1]
2. [Step 2]
3. [Step 3]

Here's why: [Reasoning]"
```

**5. Pin the discussion**
```
Click pin icon
"Great question - pinned for reference"
```

### Scenario B: Same Question Asked Multiple Times

**The Situation:**
```
Monday: Q: "How do I set up authentication?"
Wednesday: Q: "How do I add authentication?"
Friday: Q: "Authentication - how?"

[Same question, different words]
```

**Your Action:**

**1. Answer the first one comprehensively**
```
Full explanation with examples
"Pinning this - fundamental question"
```

**2. Pin it**
```
Click pin icon
Make it visible at the top
```

**3. On duplicate questions, redirect**
```
"Great question! This is answered in detail here: [pinned link]

If you have follow-up questions, reply there!"
```

**4. Consider systemic fixes**
```
If 3+ students ask this:
- Lesson content isn't clear enough
- Add clarification to lesson
- Create practice exercise
- Add to FAQ
```

### Scenario C: Student Being Disruptive

**The Situation:**
```
Student posts: "This is stupid! You're all idiots!"
Or: "You don't belong here because [identity]"
Or: Repeated attacks on another student
```

**Your Action:**

**1. Delete immediately**
```
Click [Delete] button
Choose soft or hard delete (soft is usually fine)
```

**2. Private message the student**
```
"I deleted your post because it violates community guidelines
regarding respectful communication.

Let's focus on course topics and supporting each other.
[Link to community guidelines]

If you'd like to discuss offline, I'm here to help."
```

**3. Document the incident**
```
Note in your records:
- Date/time
- What happened
- Your response
- Monitor for repeat
```

**4. If pattern continues**
```
Second violation: 3-day discussion suspension
Third violation: 7-day suspension or course removal
Escalate to admin for serious violations
```

### Scenario D: Excellent Student Contribution

**The Situation:**
```
Student posts brilliant explanation that helps everyone
Or: Student helps multiple classmates
Or: Great insights shared
```

**Your Action:**

**1. Reply with enthusiastic validation**
```
"This is fantastic, @[Student]! You've clearly understood
[concept] and explained it beautifully.

What I particularly like:
- [Specific thing 1]
- [Specific thing 2]

This is exactly the kind of peer learning we want to see!
Thanks for helping your classmates!"
```

**2. Consider pinning**
```
"Pinning this - great reference for everyone"
```

**3. Publicly celebrate**
```
In dashboard: "Shoutout to [Student] for helping the class!"
Builds community and encourages more participation
```

### Scenario E: Off-Topic Discussion

**The Situation:**
```
Students chatting about weekend plans
Or: Non-course topic getting lots of activity
Or: Discussion drifting from original topic
```

**Your Assessment:**

**Is it harmless bonding?**
- YES: Let it continue or suggest "General Discussion" forum
- NO: Redirect firmly

**Your Action (If Disruptive):**

**1. Reply with gentle redirect**
```
"I love the team spirit! However, let's keep this forum
focused on course content.

Feel free to chat in the 'General Discussion' forum or
Slack. Back to [original topic]..."
```

**2. Lock if necessary**
```
Click [Lock] button
"Thread locked - let's continue [original topic] in [new thread]"
```

**3. Optional: Create new thread**
```
Post: "Here's the continuation on [original topic]..."
Keeps discussion organized
```

## Module 5: Best Practices

### Practice 1: Be Present & Responsive

**Goal:** Create an active, engaged community

**What This Means:**
- Check discussions regularly (daily if possible)
- Respond within 24 hours
- Acknowledge good contributions
- Show you're engaged and care

**Implementation:**
```
Monday-Friday: Check discussions every morning
Weekend: Once per weekend day
Response time: 24 hours max for questions

Use calendar reminder or email notifications
```

**What Students See:**
- Teacher is paying attention
- Questions get answered promptly
- Good contributions are recognized
- Safe space to ask

### Practice 2: Encourage Peer Learning

**Goal:** Build a collaborative community

**What This Means:**
- Don't answer immediately
- Let students attempt answers
- Validate good student responses
- Create peer mentoring culture

**Implementation:**

**Pattern 1: Student asks question**
```
Day 1: Student posts question
Days 2-3: Let other students answer
Day 4: If no good answer, provide yours
         If good answer exists, validate it
```

**Pattern 2: Multiple answers**
```
Review all answers
Reply: "Great attempts, everyone! Let me add [detail]"
Validate the best student answer
Only "correct" if student answer was wrong
```

**Pattern 3: Student helps another**
```
Reply: "Amazing peer mentoring, [Student]!"
Public recognition
Encourages more peer help

Result: Students become community leaders
```

### Practice 3: Set Clear Expectations

**Goal:** Students know what's expected

**What This Means:**
- Post guidelines at course start
- Model respectful communication
- Address violations clearly
- Be transparent about rules

**Implementation:**

**Create Pinned "Guidelines" Post:**
```

‚úÖ Ask questions about course content

```

### Practice 4: Curate High-Quality Content

**Goal:** Build a knowledge resource

**What This Means:**
- Pin excellent content
- Link between related discussions
- Create summary posts
- Maintain organized structure

**Implementation:**

- 5-10 per lesson max
- High-quality answers (not quick tips)

- Essential information

```

### Practice 5: Monitor Engagement

**Goal:** Identify and help struggling students

**What This Means:**
- Track who's participating
- Notice patterns in questions
- Reach out to quiet students
- Adjust content based on confusion

**Implementation:**

**Weekly engagement check:**
```
Review last week's discussions:
- Who participated? Who didn't?
- What topics had the most questions?
- Were there patterns of confusion?
- Are specific lessons unclear?
```

**Reach out to quiet students:**
```
"Hi [Student],

I noticed you haven't posted questions yet. That's totally fine!
Just checking in - do you have questions I can help with?

You can reply publicly or message me privately.

Looking forward to hearing from you!"
```

**Adjust course if needed:**
```
If many students ask about [topic]:
- Review that lesson
- Maybe it needs clarification
- Add practice exercise
- Create supplemental resource

## Module 6: Handling Difficult Situations

### Situation A: Harassment

**What It Looks Like:**
```
"You're stupid"
"You don't belong here"
"Stop posting"
Repeated attacks on someone
```

**Your Response:**

**Immediate (within hours):**
```
1. Delete the harassment
2. Private message the student:

"Your post was deleted because it violates community
guidelines about respectful communication.

Harassment is not tolerated here.

If you have concerns about someone's posts, let me know privately.
But public attacks are not acceptable.

[Link to community guidelines]"
```

**If Pattern Emerges:**
```
After second incident:
1. Private meeting/video call
2. Document conversation
3. Clear warning: "Next violation = course removal"
4. Contact admin if third incident
```

**Support the Harassed Student:**
```
Private message:
"I saw what happened. I'm sorry you experienced that.
It's not okay, and I've addressed it.

You can always reach out to me if you need support."
```

### Situation B: Academic Dishonesty

**What It Looks Like:**
```
Student posts assignment solution
"Can someone do my homework?"
Student's code looks identical to another's
Discussion mentions exam answers
```

**Your Response:**

**First: Document**
```
Screenshot the post
Note date, time, discussion URL
Keep for records
```

**Second: Don't publicly accuse**
```
Private message:
"Hi [Student],

I noticed something in your post about [assignment].
Can you explain your approach to me?

I want to make sure you're getting the learning value
from this assignment. Can we chat?"
```

**Third: Evaluate**
```
Listen to their explanation
Do they understand their own work?
Or does it seem copied?
```

**Fourth: Take action**
```
If innocent: "Thanks for clarifying. Great work!"
If dishonest:
  "I think this might be [plagiarized/cheated].
   Let's discuss the assignment expectations and how to
   move forward."
```

**Fifth: Escalate if needed**
```
Multiple violations or refusal to cooperate:
- Contact admin
- May result in: Failed assignment, course removal, etc.
- Document everything
```

### Situation C: Crisis/Mental Health

**What It Looks Like:**
```
"I'm thinking about [self-harm]"
"I'm going to [harm myself]"
Signs of suicidal ideation
Severe anxiety/depression mentioned
```

**Your Immediate Response:**

**1. Don't ignore it**
```
This is serious - take action immediately
```

**2. Delete post (if personal crisis posted publicly)**
```
To protect privacy
Don't want others piling on
```

**3. Private message immediately**
```
"Hi [Student],

I read your recent post. I'm concerned and want to help.

You're not alone. Here are resources:
- Campus Counseling: [phone/email]
- Crisis Hotline: [number]
- Student Services: [contact]

Can I help you connect to support?

You matter, and we care about you."
```

**4. Contact student services**
```
Email student support team
Let them know you're concerned
They can follow up and offer help
```

**5. Document**
```
Save messages
Note dates and specifics
Follow school protocol for student safety
```

**6. Follow up**
```
After crisis resources engaged:
"How can I support you academically while you're going through this?
Do you need an extension or accommodations?"
```

## Module 8: Ethical Moderation

### Core Principles

**Principle 1: Consistency**
```
Apply rules the same way for everyone
Don't show favoritism
Document your decisions
Review your decisions periodically
```

**Principle 2: Transparency**
```
Explain why you delete/edit content
Don't secretly remove discussions
Admit when you're unsure
Share your moderation philosophy
```

**Principle 3: Respect**
```
Treat students fairly
Assume good intent initially
Don't moderate when angry
Take breaks if you need them
```

**Principle 4: Proportionality**
```
Match consequence to severity
First violation ‚â† permanent ban
Escalate gradually
Provide opportunities to improve
```

### Self-Check Questions

**Before you delete something, ask:**
```
1. Does this violate a clear guideline?
2. Am I being consistent with past decisions?
3. Have I given the student a chance to correct it?
4. Am I being fair?
5. Would I delete this for a student I like?
6. Can I explain this decision clearly?

If you answer "No" to any, reconsider.
```

**Before you edit someone's post, ask:**
```
1. Is this absolutely necessary?
2. Does it preserve the person's intent?
3. Does it have a clear explanation?
4. Would I want my post edited this way?
5. Is there a better way (reply instead)?

If you answer "No" to any, reconsider.
```

## Module 10: Self-Care for Moderators

### Moderator Burnout

**Signs You Need a Break:**
```
- Dreading checking discussions
- Getting frustrated easily
- Spending too much time moderating
- Not enjoying interactions
- Feeling responsible for all discussions
```

**How to Manage:**

**Set Boundaries:**
```
- Check discussions at set times (not constantly)
- Take weekends off
- Don't respond to every post
- Assign peer moderators if possible
```

**Delegate:**
```
- Ask advanced students to help moderate
- Create peer mentor roles
- Post guidelines so students self-moderate
```

**Take Care of Yourself:**
```
- Take days off
- Talk to other teachers
- Seek support if handling serious issues
- Remember: You can't help everyone
```

**It's OK to:**
```
- Not answer every question immediately
- Not respond on weekends
- Ask for help with moderation
- Set "office hours" for questions
- Decline to moderate certain topics
```

## Quiz: What You've Learned

**Test Your Knowledge:**

**1. What should you do when a student posts misinformation?**
- A) Immediately delete it
- B) Reply to correct it and validate the question
- C) Message the student privately
- D) Ignore it

**Answer: B** - Correct misinformation by replying to the original question. Show why the information is wrong and provide the right answer.

**2. When should you pin a discussion?**
- A) Whenever a student asks a question
- B) When it's a comprehensive answer to a commonly-asked question
- C) When you've replied to it
- D) Never - let students find answers themselves

**Answer: B** - Pin discussions that provide high-quality answers to frequently-asked questions.

**3. What's the best approach when a student asks for assignment answers?**
- A) Delete their post
- B) Reply with the answer
- C) Guide them with hints instead
- D) Contact admin

**Answer: C** - Guide students to the answer rather than providing it directly. This maintains academic integrity while helping them learn.

**4. How often should you check discussions?**
- A) Every hour
- B) Once per week
- C) Daily (or multiple times per day)
- D) Only when students complain

**Answer: C** - Check discussions daily to stay responsive and engaged. This builds community and ensures questions get answered promptly.

**5. What's a sign you should moderate more strictly?**
- A) One student made a small mistake
- B) Pattern of violations by same student
- C) A question you don't like
- D) A student disagrees with you

**Answer: B** - Look for patterns of violations. First violations deserve warning/gentle correction. Patterns indicate need for stricter action.
